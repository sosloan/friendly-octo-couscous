HOW THIS WORKS
--------------
1. Pick any prompt below. Copy it. Paste it. Hit send. That is 1-click and go.
2. After you read the response, look at the three "NEXT PROMPTS" lines.
3. Pick one. That becomes your next 1-click prompt.
4. The DO LESS line is a negative signal -- it tells you when to pull back.
5. The gallery below is Gallery 0 (the starting 3x3).
   As you move through the feed-forward chain, a new 3x3 emerges each time.

================================================================================
 GALLERY NAVIGATION RULES
================================================================================

  PICK -> RESPOND -> CHOOSE NEXT -> REPEAT

  Every response you get back automatically surfaces 3 forks:

    [+] DO MORE  = Add complexity, stakes, or scope. Level up.
    [=] STAY     = Same difficulty, fresh scenario. Refresh the gallery tile.
    [-] DO LESS  = Reduce scope. Nail the core thing. This is your STOP signal.
                   If every response is saying DO LESS, you found the right
                   layer of granularity. Park there and build depth, not width.

  THE 3x3 UPDATES WHEN:
    * You choose DO MORE three times in a row -> New Gallery spawned.
    * You choose STAY on any row -> That row's prompts rotate.
    * You choose DO LESS -> Gallery stays, but that prompt locks simpler.

================================================================================
 FEED-FORWARD SYSTEM SUMMARY
================================================================================

  Each prompt is a node. Each response is an edge. The gallery is a graph.

  The three branches per node mean the gallery is always 3 moves deep.
  You never hit a dead end. You either go deeper ([+]), sideways ([=]),
  or you find the floor ([-]) -- and the floor is useful too.

  Starting nodes in Gallery 0:   A1, A2, A3, B1, B2, B3, C1, C2, C3
  Gallery 1 nodes:               A4, B4, C4 (+ all [+] forks from Gallery 0)
  Gallery 2+:                    Generated from [+] chains in Gallery 1

  The Sheriff's team can run this like a menu.
  One person's [+] fork is another person's A1.
  No dead ends. No "what should I ask?" paralysis.
  The prompts are already written. Pick one. Go.
